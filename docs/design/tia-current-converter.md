# Theory of Operation: TIA Current Converter

**Responsibility:** Convert the raw 16-bit ADC value into a physically meaningful current in milliamps (mA), representing the light intensity received by the CNY70 sensor.

---

## 1. Context: What Does the ADC Value Represent?

The CNY70 is an optical reflectance sensor: it emits infrared light and measures the intensity of the light reflected back by the hammer shank. The closer the shank, the more light is reflected, and the higher the photocurrent generated by the sensor.

This current is not read directly by the MCU. It first passes through a **Transimpedance Amplifier (TIA)** stage, which converts it into a voltage readable by the ADC.

### 1.1 The TIA Stage

The TIA is built around a TLV9004S operational amplifier with a feedback resistor $R_f = 1800\ \Omega$. It operates as an **inverting stage** relative to the voltage reference $V_{REF} = 2.048\ \text{V}$ (provided by a REF3020 precision reference):

$$V_{out} = V_{REF} - (I_{in} \times R_f)$$

As sensor current increases (shank getting closer), $V_{out}$ decreases. The ADC then digitizes this voltage over a 16-bit range referenced to $V_{REF}$.

### 1.2 The Inversion Effect

This inverting topology has a direct consequence on the raw ADC value:

| Sensor State | ADC Value | Voltage at MCU | Input Current |
| :--- | :--- | :--- | :--- |
| **No light** (hammer far) | 65535 | 2.048 V | 0 µA |
| **Mid range** | ~32768 | ~1.024 V | ~569 µA |
| **Full light** (hammer close) | 0 | 0 V | ~1.138 mA |

A **high ADC value means low current** (hammer far from sensor), and a **low ADC value means high current** (hammer close). This inversion must be accounted for before any further processing.

---

## 2. Why Convert to Current?

The raw ADC value is an artifact of the electronic chain: it encodes both the sensor signal and the specific parameters of the acquisition circuit ($V_{REF}$, $R_f$, ADC resolution). Two boards with different feedback resistors would produce different ADC values for the same physical situation.

Converting to current in mA produces a value that:
- Is **independent of the circuit parameters**, making it portable across hardware revisions.
- Is the **natural physical quantity** produced by the CNY70 photodiode.
- Serves as the correct input for the linearization stage, which maps current to normalized position based on the sensor's physical response curve.

---

## 3. Conversion Formula

### 3.1 General Form

Inverting the TIA equation and expressing the result in mA:

$$I_{in}\ (\text{mA}) = \frac{V_{REF} - V_{measured}}{R_f}$$

Since $V_{measured} = \frac{ADC\_val}{65535} \times V_{REF}$, this becomes:

$$I_{in}\ (\text{mA}) = \frac{(65535 - ADC\_val) \times V_{REF}}{65535 \times R_f}$$

### 3.2 Optimized Form

To avoid repeated floating-point divisions at runtime, a scale factor $S_f$ is computed once at initialization:

$$S_f = \frac{V_{REF}}{65535 \times R_f}$$

The runtime conversion then reduces to a single multiplication:

$$I_{in}\ (\text{mA}) = (65535 - ADC\_val) \times S_f$$

The subtraction $(65535 - ADC\_val)$ corrects for the inverting topology, ensuring the output is a positive, monotonically increasing value as the hammer approaches the sensor.

---

## 4. Implementation

The `TiaCurrentConverter` stage is parameterized by three compile-time constants:

- `ADC_REFERENCE_VOLTAGE_MILLI_VOLTS`: $V_{REF}$ in millivolts (2048 mV).
- `ADC_RESOLUTION_BITS`: ADC bit depth (16 bits, range 0–65535).
- `TIA_FEEDBACK_RESISTOR_OHMS`: $R_f$ in ohms (1800 Ω).

**Input:** Raw 16-bit ADC value (0–65535).

**Output:** Sensor current in mA — a positive value, increasing as the hammer shank approaches the sensor — passed to the `AnalogSensorLinearizer` stage.
